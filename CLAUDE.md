## Commands

| Command | Description |
|---------|-------------|
| `make generate` | Regenerate `api/types.gen.go` and `api/server.gen.go` from the OpenAPI spec. Required before building or testing after any spec change. |
| `make build` | Build both binaries (`entitlements-api-go` and `bundle-sync`). |
| `make test` | Run all tests with verbose output (runs `make generate` first). |
| `make test-all` | Run all tests with race detector, serial execution (`-p 1`), and coverage (runs `make generate` first). This is what CI runs. |
| `make bench` | Run benchmarks. |
| `make debug-run` | Run the server locally with `ENT_DEBUG=1` (mock AMS/BOP — local dev only, never production). |
| `make image` | Build the container image with podman. |

## CI Checks

- **Konflux/Tekton** (`.tekton/`): Primary CI gate. Runs `make generate` then `make test-all` with race detector on every PR. A PR cannot merge if this fails.
- **GitHub Actions** — `json-yaml-validate`: Validates all JSON and YAML files (excluding `err_bundle.yml`). Runs on every PR. Ensure any new YAML/JSON files are valid.
- **GitHub Actions** — security scan: Grype/Syft container image vulnerability scan. Does not run Go tests.

## Behavioral Notes

- Never edit or commit `*.gen.go` files — they are gitignored and regenerated by `make generate`.
- Run `make generate` after cloning (generated files are not committed) and after any change to `apispec/api.spec.json`.
- The PR template includes a secure coding checklist. Review and address relevant items when opening a PR.
- `bundles/bundles.yml` is gitignored. Only `bundles/bundles.example.yml` is committed. Do not commit real bundle config.
